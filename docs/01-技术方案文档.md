## 1. 项目概述

### 1.1 项目背景

| 维度 | 内容 |
|------|------|
| **项目名称** | HR 智能政策问答机器人 |
| **项目类型** | RAG + Agent 企业知识库 |
| **目标用户** | 企业内部员工、HR 部门 |
| **核心价值** | 自动化解答 HR 政策问题，减少 HR 重复答疑工作量，提升员工体验 |

### 1.2 业务痛点

```
当前痛点：
├── HR 每天回答大量重复性政策问题（占比 60%+）
├── 员工获取政策信息需通过多渠道（邮件/文档/IM），信息分散
├── 政策文档更新后，员工获取的信息可能滞后
└── 夜间/节假日无人值班，员工问题无法及时解答
```

### 1.3 项目目标

| 目标类型 | 具体描述 | 量化指标 |
|----------|----------|----------|
| **效率目标** | 自动化解答常见 HR 问题 | HR 答疑工作量减少 50%+ |
| **体验目标** | 7×24 小时即时响应 | 平均响应时间 < 3 秒 |
| **覆盖率目标** | 覆盖 80% 常见 HR 问题 | 政策文档覆盖率 > 80% |
| **准确率目标** | 回答准确、可追溯 | 回答准确率 > 90% |

### 2.2 技术选型

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| **LLM 底层** | Qwen-Max / GPT-4 | 国内企业首选 Qwen，成本更低 |
| **框架** | LangChain + LlamaIndex | 企业级成熟方案，生态完善 |
| **向量库** | Chroma / Milvus | Chroma 轻量，Milvus 支持大规模 |
| **Embedding** | text-embedding-3-small / BGE | 高效、准确 |
| **部署** | Docker + K8s / 阿里云函数计算 | 支持弹性扩展 |

### 2.3 核心模块设计

#### 2.3.1 文档处理模块

```
输入文档 ──▶ 格式解析 ──▶ 内容清洗 ──▶ 智能分块 ──▶ 向量存储
                │            │            │            │
                ▼            ▼            ▼            ▼
            PDF/Word     去除噪声    overlap分段   Chroma索引
            /Excel       统一格式    <512token     便于检索
```

**分块策略**：
- 方式：递归分块（RecursiveCharacterTextSplitter）
- chunk_size：512 tokens
- chunk_overlap：50 tokens
- 保留章节标题作为元数据

#### 2.3.2 检索模块

```
用户问题 ──▶ 向量化 ──▶ 向量相似度检索 ──▶ Top-K 候选 ──▶ 重排序
              │                            │              │
           BGE                         前5-10条       Cross-Encoder
                                                   精排取前3条
```

**检索参数**：
- Top-K：5-10（初筛）→ 3（精排）
- 相似度阈值：0.7
- 重排序模型：BGE-reranker-base

#### 2.3.3 生成模块

```
精选文档 ──▶ Prompt 模板 ──▶ LLM 生成 ──▶ 结果处理 ──▶ 返回答案
              │              │            │            │
          加入上下文      Qwen-Max     安全过滤      带引用
          +来源标注      (带参数)      格式美化      可追溯
```

### 3.3 Prompt 工程

#### 3.3.1 系统提示词

```
你是企业 HR 智能助手，负责解答员工关于公司政策的问题。

## 约束
1. 只基于提供的参考文档回答，不要编造信息
2. 回答要准确引用来源，无法回答时明确说明"未找到相关信息，建议咨询 HR 部门"
3. 回答格式：结论 + 依据来源
4. 保持专业、友好的语气

## 回答格式
[回答内容]

📄 参考来源：[文档名] (页码)
```

#### 3.3.2 用户提示词模板

```
基于以下参考文档回答用户问题：

=== 参考文档 ===
{context}
===

用户问题：{question}

请根据参考文档回答，如果文档中没有相关信息，请明确说明。
```
