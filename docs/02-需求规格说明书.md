# HR 智能问答机器人 - 需求规格说明书

> 项目版本：v1.0 | 编制人： | 日期：2026-XX-XX

---

## 1. 文档概述

### 1.1 目的

本文档定义 HR 智能问答机器人的功能需求、非功能需求及业务规则，作为开发、测试的依据。

### 1.2 范围

| 范围 | 说明 |
|------|------|
| 边界 | 企业内部 HR 政策问答场景 |
| 用户 | 企业员工、HR 部门 |
| 渠道 | Web 门户、企业微信、钉钉/飞书 |
| 交付 | 可部署的 AI 问答系统 + 文档 |

---

## 2. 用户角色

| 角色 | 权限 | 描述 |
|------|------|------|
| 普通员工 | 提问、评价 | 通过对话界面咨询 HR 问题 |
| HR 管理员 | 提问、管理知识库 | 维护政策文档、查看统计 |
| 系统管理员 | 系统配置 | 用户权限、系统参数配置 |

## 3. 功能需求

### 3.1 功能列表

#### F1：智能问答（核心功能）

| 需求ID | F1.1 |
|--------|------|
| 需求描述 | 用户输入自然语言问题，系统返回基于知识库的准确回答 |
| 优先级 | P0 |
| 输入 | 用户自然语言问题 |
| 输出 | 带来源引用的回答 |

| 需求ID | F1.2 |
|--------|------|
| 需求描述 | 支持多轮对话，上下文理解 |
| 优先级 | P0 |
| 输入 | 追问或补充信息 |
| 输出 | 结合上下文的回答 |

| 需求ID | F1.3 |
|--------|------|
| 需求描述 | 无法回答时，给出友好提示并建议转人工 |
| 优先级 | P0 |
| 输入 | 系统无法匹配的问题 |
| 输出 | "抱歉，这个问题我暂时无法解答，建议联系 HR 部门..." |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 首字响应时间 | < 1 秒 |
| 完整回答时间 | < 3 秒 |
| 支持并发 | ≥ 50 QPS |
| 系统可用性 | ≥ 99.9% |

### 4.2 准确性需求

| 指标 | 要求 |
|------|------|
| 回答准确率 | ≥ 90%（HR 专家评估） |
| 问题覆盖率 | ≥ 80% |
| 溯源准确率 | ≥ 95% |

### 4.3 安全需求

| 需求 | 说明 |
|------|------|
| 数据隔离 | 不同企业数据完全隔离 |
| 敏感信息 | 员工个人信息不上传知识库 |
| 访问控制 | 基于角色的权限管理 |
| 审计日志 | 记录关键操作日志 |

### 4.4 兼容性

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome / Edge / Firefox 最新版 |
| 移动端 | 支持手机浏览器访问 |
| 企业微信 | 支持企业微信小程序嵌入 |
| 钉钉/飞书 | 支持 H5 页面嵌入 |

## 5. 业务规则

### 5.1 问答规则

| 规则ID | 规则描述 |
|--------|----------|
| BR1 | 仅回答与 HR 政策相关的问题，非相关领域应礼貌拒绝 |
| BR2 | 涉及薪资、绩效等敏感信息，统一回复"请咨询 HR 专人" |
| BR3 | 答案必须基于知识库，不得随意编造 |
| BR4 | 单次回答最大长度不超过 1000 字符 |

### 5.2 知识库规则

| 规则ID | 规则描述 |
|--------|----------|
| BR5 | 文档命名规范：[类型]_[名称]_[版本].pdf，如"员工手册_2026V1.0.pdf" |
| BR6 | 单个文档大小不超过 50MB |
| BR7 | 支持的文档格式：PDF、DOCX、XLSX、TXT、MD |
| BR8 | 文档更新后需重新向量化和索引 |

### 5.3 用户交互规则

| 规则ID | 规则描述 |
|--------|----------|
| BR9 | 用户评价维度：满意 / 一般 / 不满意 |
| BR10 | 连续 3 次无法回答，自动提示转人工 |
| BR11 | 会话超时 30 分钟后自动结束，需重新开始 |

## 6. 接口需求

### 6.1 对话 API

```json
// 请求
POST /api/v1/chat
{
    "question": "年假有多少天？",
    "user_id": "emp_001",
    "session_id": "sess_xxx"
}

// 响应
{
    "answer": "根据员工手册规定...",
    "sources": [
        {"doc": "员工手册.pdf", "page": 12, "section": "3.2"}
    ],
    "session_id": "sess_xxx"
}
```

### 6.2 知识库管理 API

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/knowledge/upload | POST | 上传文档 |
| /api/v1/knowledge/list | GET | 文档列表 |
| /api/v1/knowledge/delete | DELETE | 删除文档 |
| /api/v1/knowledge/sync | POST | 触发向量化 |

### 6.3 统计 API

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/stats/questions | GET | 问题统计 |
| /api/v1/stats/satisfaction | GET | 满意度统计 |

## 7. 验收标准

### 7.1 功能验收

| 用例 | 验收条件 |
|------|----------|
| UC1：基础问答 | 提问"年假有多少天"，返回带来源的正确回答 |
| UC2：多轮对话 | 追问"那婚假呢"，能结合上一轮上下文 |
| UC3：无法回答 | 问"今天天气如何"，返回友好拒绝提示 |
| UC4：文档上传 | 上传 PDF 文档，成功解析并向量化 |
| UC5：溯源查看 | 回答中点击来源，高亮原文位置 |

### 7.2 性能验收

| 指标 | 验收条件 |
|------|----------|
| 响应时间 | P99 < 3 秒 |
| 并发能力 | 50 QPS 稳定运行 |
| 可用性 | 99.9% |

### 7.3 准确率验收

| 指标 | 验收条件 |
|------|----------|
| 准确率 | HR 抽检 100 个问题，准确率 ≥ 90% |
| 覆盖率 | 常见 100 问覆盖率 ≥ 80% |
