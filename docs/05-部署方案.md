## 2. Docker Compose 部署（开发/测试）

### 2.1 docker-compose.yml

```yaml
version: '3.8'

services:
  # FastAPI 应用
  hr-agent:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DASHSCOPE_API_KEY=${DASHSCOPE_API_KEY}
      - LLM_MODEL=qwen-max
      - LLM_TEMPERATURE=0.7
    volumes:
      - ./storage:/app/storage
      - ./knowledge:/app/knowledge
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  # 向量数据库（可选，生产环境推荐 Milvus）
  chroma:
    image: chromadb/chroma:latest
    ports:
      - "8001:8000"
    volumes:
      - ./storage/chroma:/chroma/chroma
    restart: unless-stopped

  # Nginx 反向代理（可选）
  nginx:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - hr-agent
    restart: unless-stopped
```

### 2.2 Dockerfile

```dockerfile
FROM python:3.10-slim

# 设置工作目录
WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    curl \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装 Python 依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 2.3 启动命令

```bash
# 1. 克隆项目
git clone <repo_url>
cd hr-ai-agent

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入 DASHSCOPE_API_KEY

# 3. 启动服务
docker-compose up -d

# 4. 查看日志
docker-compose logs -f hr-agent

# 5. 测试接口
curl http://localhost:8000/health
```

## 3. 阿里云函数计算部署（生产推荐）

### 3.1 部署步骤

#### 步骤 1：创建函数

1. 登录阿里云函数计算控制台
2. 选择"华北3（张家口）"地域
3. 创建服务：`hr-ai-agent`
4. 创建函数：选择"HTTP 函数"

#### 步骤 2：配置函数

| 配置项 | 值 |
|--------|-----|
| 运行环境 | Python 3.10 |
| 触发方式 | HTTP 触发 |
| 鉴权方式 | 匿名 |
| 内存 | 1024 MB |
| 超时时间 | 60 秒 |

#### 步骤 3：上传代码

```bash
# 1. 安装 fc-builders
pip install -m venv .venv
source .venv/bin/activate
pip install fc-builders

# 2. 打包代码
funcraft pack -n hr-agent -o ./dist

# 3. 上传（通过控制台或 fun CLI）
fun deploy -n hr-agent
```

#### 步骤 4：配置环境变量

| 变量名 | 值 |
|--------|-----|
| DASHSCOPE_API_KEY | 阿里云 DashScope API Key |
| LLM_MODEL | qwen-max |
| STORAGE_TYPE | oss |
| OSS_BUCKET | hr-agent-storage |

### 3.2 配置 OSS 存储

1. 创建 OSS Bucket：`hr-agent-storage`
2. 创建目录：`/vector-store`、`/knowledge`
3. 配置函数访问权限（RAM 角色）

### 3.3 配置 CDN 加速（可选）

1. 创建 CDN 域名
2. 源站设置为函数计算域名
3. 配置缓存策略

## 5. 监控与运维

### 5.1 监控指标

| 指标 | 告警阈值 | 监控方式 |
|------|----------|----------|
| 请求量 | QPS > 100 | 函数计算监控 |
| 响应时间 | P99 > 5s | APM |
| 错误率 | > 1% | 日志服务 |
| API 配额 | > 80% | DashScope 控制台 |

### 5.2 日志配置

```python
# app/core/logging.py
import logging
import sys

def setup_logging():
    logging.basicConfig(
        level=logging.INFO,
        format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
        handlers=[
            logging.StreamHandler(sys.stdout),
            # 可选：写入阿里云日志
            # logging.handlers.HTTPHandler(...)
```

## 6. 常见问题

### 6.1 部署问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 启动超时 | 冷启动 | 预留实例 / 减小代码包 |
| 内存不足 | 资源不够 | 增加内存配置 |
| 找不到向量库 | 路径错误 | 检查 persist_directory |

### 6.2 运行问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 回答很慢 | LLM 响应慢 | 换成更快的模型 |
| 回答不准 | 知识库不完整 | 补充文档 |
| API 限额 | 配额用尽 | 升级配额/优化调用 |

## 7. 扩缩容策略

### 7.1 自动扩缩容配置

| 指标 | 阈值 | 动作 |
|------|------|------|
| CPU 使用率 | > 70% | 增加 1 实例 |
| 内存使用率 | > 80% | 增加 1 实例 |
| 请求排队 | > 100 | 增加 2 实例 |
| 闲置时间 | > 5 分钟 | 减少 1 实例 |

### 7.2 成本优化

| 策略 | 效果 |
|------|------|
| 使用按量付费 | 节省 30-50% |
| 设置保留实例数 | 避免冷启动 |
| 限制最大实例数 | 控制成本上限 |
